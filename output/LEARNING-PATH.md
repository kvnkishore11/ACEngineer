## ‚ö†Ô∏è Common Pitfalls & Solutions

### Pitfall 1: Trying to Skip Foundations
**Problem**: Jumping to advanced modules without basics
**Solution**: TAC-1-3 concepts appear everywhere; master them first

### Pitfall 2: Over-Engineering Simple Problems
**Problem**: Using complex patterns for simple tasks
**Solution**: Start with Minimum Viable, scale when needed

### Pitfall 3: Ignoring Context Limits
**Problem**: Agents failing due to context overflow
**Solution**: Apply R&D framework early and often

### Pitfall 4: Monolithic Agent Design
**Problem**: Creating do-everything agents
**Solution**: Specialize agents for specific tasks

### Pitfall 5: Neglecting Error Handling
**Problem**: Agents failing silently
**Solution**: Build robust error recovery patterns

### Pitfall 6: Poor Documentation
**Problem**: Can't understand system after building
**Solution**: Use self-documenting patterns from TAC-6

### Pitfall 7: Skipping Testing
**Problem**: Unreliable agent behaviors
**Solution**: TAC-5 testing patterns are non-negotiable

### Pitfall 8: Premature Optimization
**Problem**: Optimizing before understanding needs
**Solution**: Build, measure, then optimize
